# Codex Style

When editing code:

- Inspect before modifying.
- Preserve existing user changes.
- Prefer existing project conventions.
- Avoid unrelated refactors.
- Keep code comments rare and useful.
- Run focused validation after changes.
- Summarize changed files and verification results.

When creating a new project:

- Start with the actual app/tool, not a marketing landing page.
- Include a README with setup and run commands.
- Include tests when the language stack has a normal test runner.
