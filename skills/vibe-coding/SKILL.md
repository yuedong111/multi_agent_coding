# Vibe Coding

Use this rhythm:

1. Define outcome, user, and success criteria.
2. Create a compact plan: files, modules, responsibilities, verification.
3. Build one small increment at a time.
4. Keep boundaries clear. Do not implement future steps early.
5. Every code step should have a verification step.

Prefer:

- Clear file structure before code volume.
- Small modules with single responsibility.
- Explicit interfaces.
- Minimal edits that preserve future flexibility.
- Recoverable task state.
